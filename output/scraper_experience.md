# 亲子活动数据搜集经验总结

> 10城5000+真实亲子活动数据搜集方法论
> 2026年7月

---

## 一、核心结论

**全部真实活动的搜集，WebSearch + 人工整理的组合是最高效的路径。** 直接用Python爬虫从各大官网抓数据困难重重（动态渲染、SSL证书、API鉴权、反爬机制），而通过搜索引擎找到汇总文章/官方公告，再批量提取，效率更高且数据质量更有保障。

---

## 二、走过的弯路

### 1. 直接爬虫行不通的场景

| 方案 | 问题 | 结论 |
|------|------|------|
| 国家公共文化云 | 页面动态渲染，数据通过JS异步加载，静态爬虫抓不到 | ❌ 放弃 |
| 本地宝 | 爬到的大部分是演唱会/商业演出新闻，不是亲子活动 | ❌ 放弃 |
| 博物馆官网 | SSL证书问题（BAD_ECPOINT），Python requests无法请求 | ⚠️ 需用curl绕过 |
| 成都文化天府云API | 需要token和appcode，鉴权逻辑复杂 | ❌ 放弃 |
| 多城市通用爬虫框架 | 每个城市数据源不同，通用框架复用率低 | ❌ 投入产出比低 |

### 2. 虚假数据的教训

- 早期用模板批量生成了5000+"虚假活动"，全部被要求删除
- 场馆名用通用名、活动名用模板式描述，一眼就能看出是假的
- **核心教训：宁可数量少，也不能有假数据。用户对真实性零容忍。**

## 三、高效搜集方法论

### 策略："汇总文章提取法"

与其一个个场馆去搜，不如搜"XX市暑期亲子活动汇总"类文章，一篇文章就能提取30-50个活动。

#### 搜索关键词模板（按效率排序）

1. **汇总攻略类**（效率最高，一篇顶十篇）
   - `{城市} 2026年暑假 亲子活动 汇总`
   - `{城市} 暑期亲子活动攻略 2026`
   - `{城市} 暑期 100个 亲子活动`
   - `{城市} 暑期文旅消费季 活动`
   - `{城市} 消夏 文旅活动 汇总`

2. **官方渠道类**（信息最权威）
   - `{城市} 文化和旅游局 暑期活动 2026`
   - `{城市} 文旅局 暑假 活动`
   - `{城市} 文化馆 暑期 2026`
   - `{城市} 图书馆 暑期活动 2026`

3. **场馆聚焦类**（适合深挖补充）
   - `{城市} 博物馆 暑期活动 2026`
   - `{城市} 科技馆 暑期 2026`
   - `{城市} 少年宫 暑期 2026`
   - `{城市} 儿童剧 演出 2026年7-8月`

4. **区县下沉类**（大城市深挖时使用）
   - `{区名} 暑期活动 2026`
   - `{区名} 文化馆 暑期 少儿 2026`
   - `{区名} 图书馆 暑期 亲子 2026`

#### 搜集节奏（以500条/城为目标）

| 轮次 | 策略 | 预计产出 | 累计 |
|------|------|---------|------|
| 第1轮 | 全网广撒网（9大类方向各搜2-3次） | 80-120条 | 100条 |
| 第2轮 | 重点场馆深挖（博物馆/科技馆/图书馆/少年宫/剧院/乐园各搜） | 50-100条 | 200条 |
| 第3轮 | 汇总攻略文章批量提取（找5-10篇汇总文） | 100-180条 | 350条 |
| 第4轮 | 各区县下沉（10+个区县各搜活动） | 100-200条 | 500条+ |

## 四、数据质量把控

### 活动数据10字段标准

```
title              # 必须有具体名称，不能是"暑期活动"这种泛称
venue              # 真实场馆/地点全称
city               # 城市代码（英文）
start_date/end_date # YYYY-MM-DD格式，至少精确到月
link               # 真实来源URL
description        # 30字以上，描述活动内容
fee                # 免费/免费需预约/收费/部分免费
source             # 数据来源
family_friendly    # true
category           # 自动分类（展览/科普/演出/阅读/体育等）
```

### 去重规则

按 `(title, venue, start_date)` 三元组去重。

### 常见数据问题

| 问题 | 处理方式 |
|------|---------|
| 日期模糊（"7月-8月"） | 标准化为当月首日-末日 |
| 费用描述多样（"公益免费"/"免预约"/"凭票入场"） | 统一归为4类：免费/免费需预约/收费/部分免费 |
| 活动描述过短（<10字） | 用"title，在venue举办。"句式补充 |
| 活动描述过长（>500字） | 截断 |
| 分类不明 | 按标题关键词自动分类 |

## 五、敏感词合规经验

### 词库分类

8大类敏感词：政治敏感、暴力恐怖、色情低俗、毒品、赌博、迷信诈骗、地区主权、广告营销。

### 白名单机制（关键！防误判）

两级白名单：
1. **SIMPLE_WHITELIST**：整词匹配，文本包含即豁免（如"鸦片战争"、"微信公众号"）
2. **WHITELIST_MAP**：针对特定敏感词的白名单（如"法轮"对应"法轮寺/法轮殿"等）

### 典型误判案例

| 敏感词 | 误判场景 | 解决方案 |
|--------|---------|---------|
| 鸦片 | 鸦片战争（历史名词） | 加入白名单 |
| 微信公众号 | 活动报名渠道 | 加入白名单 |
| SM | 上海缩写/品牌名 | 直接从词库移除（过短，误判率太高） |
| 网络诈骗 | 法治宣传案例 | 加入白名单 |
| 法轮 | 法轮寺/法轮殿（佛教建筑） | 映射白名单 |
| 民运 | 农民运动讲习所（革命场馆） | 映射白名单 |

### 脱敏方案

检测到敏感词时，用 `首尾字符 + 中间*` 的方式脱敏（如"台独"→"台*"，"法轮功"→"法*功"）。

## 八、后续优化方向

1. **自动化抓取**：针对已验证可爬的网站（如深圳本地宝），写定向爬虫定时增量更新
2. **数据质量评分**：给每条活动打质量分（有具体日期/有链接/有详细描述=高分）
3. **过期自动清理**：活动结束后自动从主列表移除，归档到历史库
4. **场馆-活动关联增强**：活动自动匹配venue_info中的场馆，补充场馆信息
5. **用户反馈机制**：小程序端增加"举报不实"按钮，用户可反馈虚假活动
