# 数据采集经验总结（供复用）

> 最后更新：2026-07-11
> 本文档记录各场馆数据采集方式、URL规律、注意事项，供后续数据更新复用。
> 覆盖城市：深圳（已上线）、广州、北京、上海、杭州（规划中）

---

## 一、采集策略总览

### 核心原则
1. **真实可溯源** — 每条活动必须来自官方渠道（官网/公众号/政府网站），保留原始链接
2. **交叉验证** — 官网无法直接抓取时，通过搜索引擎+媒体报道+票务平台交叉验证
3. **时间聚焦** — 重点采集未来1-2个月的活动，过期数据及时清理
4. **亲子优先** — 项目定位亲子活动日历，优先收录 family_friendly 活动

### 采集流程
```
官网首页 → 活动/展览列表页 → 逐条提取活动信息
    ↓（失败）
搜索引擎 "场馆名+年月+活动" → 媒体报道/公众号文章 → 提取信息
    ↓（补充）
票务平台（大麦/猫眼/大河票务）→ 票价/演出时间补充
```

## 二、政务公开数据渠道（优先用，最权威）

### 核心结论

1. **公办场馆100%强制公开** — 图书馆/博物馆/科技馆/少年宫/文化馆/公共体育中心等国有公共文化场馆，其名录、地址、开放时间、活动预告均属政务公开法定内容，必须在市文旅局、市体育局、政务数据开放平台公示。
2. **两层渠道结构**：
   - **市级文旅官网**：汇总本市所有公办文体场馆（最精准，含地址、电话、官网）
   - **国家公共文化云**：全国通用平台，可跨城市检索场馆+活动
3. **商业综合体不在政府统一收录范围内** — 欢乐港湾、前滩太古里、永庆坊这类商业滨水综合体，属于商业配套，只能在商场官方公众号/小程序获取活动。

### 数据权威性排序

```
政府官网名录 > 场馆独立官网 > 国家公共文化云 > 商业公众号
```

### 采集工具实操建议

1. **场馆基础数据**：优先抓取市文旅局官网场馆名录页面，一次性导出全城市公办场馆基础数据（名称、地址、官网）
2. **活动数据**：同步对接「国家公共文化云」+ 各场馆独立官网活动栏目，抓取免费亲子公益活动
3. **商业综合体**：单独建库，无法通过政府统一页面批量获取，需单独收录
4. **数据优先级**：政府官网名录 > 场馆独立官网 > 国家公共文化云 > 商业公众号

## 四、通用技巧

### 1. 搜索引擎关键词模板
```
"{场馆名}" "{年份}年{月份}月" 活动
"{场馆名}" 暑期 活动
"{场馆名}" 展览 最新
```

### 2. 媒体来源可靠性排序
1. **官方** — 场馆官网 > 官方公众号 > 政府网站
2. **媒体** — 深圳新闻网 > 深圳本地宝 > 搜狐号(官方号) > 腾讯新闻
3. **票务** — 大麦 > 猫眼 > 大河票务 > 携程
4. **UGC** — 大众点评/小红书（仅参考，不作数据源）

### 3. WebFetch 使用要点
- 很多政府网站(.gov.cn)有SSL证书问题，需禁用验证
- JS动态加载的列表页WebFetch无法获取，需找API或备用入口
- 微信公众号文章可通过WebFetch直接获取内容
- 部分网站有反爬，需配合搜索引擎缓存

### 4. 数据字段规范
```json
{
  "title": "活动名称",
  "venue": "场馆名称（与config.py一致）",
  "start_date": "YYYY-MM-DD",
  "end_date": "YYYY-MM-DD",
  "link": "原始来源链接（必须保留）",
  "description": "活动描述（含时间/地点/嘉宾/报名方式）",
  "category": "展览/演出/讲座阅读/科普活动/亲子活动/体育赛事/影视放映/其他",
  "fee": "免费/免费需预约/收费",
  "contact": "联系电话（如有）",
  "family_friendly": true/false,
  "source": "数据来源说明"
}
```

### 5. 常见问题与解决

| 问题 | 解决方案 |
|------|---------|
| 官网打不开 | 换搜索引擎缓存/媒体转载 |
| 活动列表JS渲染 | 找API接口（F12 Network）/用搜索引擎"site:域名 活动" |
| 日期显示偏移 | 部分网站系统年份+1，需根据活动内容/季节判断真实年份 |
| 票价缺失 | 搜票务平台（大麦/猫眼）补充 |
| 活动已过期 | 保留但标注，下次更新时清理 |

## 十一、多城市扩展规划

### 扩展优先级
1. **第一梯队**：广州（邻近深圳，岭南文化相近）
2. **第二梯队**：上海（亲子活动资源丰富）
3. **第三梯队**：北京（首都，场馆最多）
4. **第四梯队**：杭州（新一线城市，文化氛围好）

### 每城市最低配置（参考深圳）
- 市级图书馆 × 1-2
- 少年儿童图书馆 × 1
- 市级博物馆 × 1-2
- 科技馆/科学中心 × 1
- 文化馆/群艺馆 × 1
- 青少年宫 × 1
- 区级图书馆 × 3-5
- 特色博物馆/美术馆 × 3-5

### 数据字段扩展
需在现有数据结构基础上增加 `city` 字段：
```json
{
  "city": "guangzhou",
  "district": "天河区",
  ...
}
```
城市代码：`shenzhen` / `guangzhou` / `shanghai` / `beijing` / `hangzhou`
