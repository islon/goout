## v1.0 — 初版发布（2026-07-11）

将 https://islon.github.io/goout/ H5 网页项目完整转换为微信小程序。

### 新增功能
- **活动列表页**：488 条活动数据，支持城市、时间、亲子、类型、区县、地点、费用共 7 个维度筛选
- **智能筛选**：无结果选项自动置灰，避免空筛选
- **搜索功能**：支持活动名称、地点关键词搜索
- **分页加载**：每页 20 条，支持上一页/下一页
- **活动详情页**：点击卡片跳转详情，展示完整活动信息和原始链接
- **关于页**：项目介绍、覆盖城市、日历订阅链接
- **日期标记**：今天/明天/即将开始的状态 Badge
- **亲子 & 费用标识**：亲子友好、免费/收费可视化标签

### 技术实现
- 数据内嵌为 JS 模块（488 条，约 320KB）
- 筛选逻辑完整复用原网页 JavaScript
- scroll-view 实现横向滚动筛选条
- 详情页通过 encodeURIComponent 传递活动 ID
- 沙色系（#D4A373）设计风格，与原网页保持一致

## v2.0 — 功能增强（2026-07-11）

### 新增功能
- **筛选器展开优化**：地点筛选"展开更多"从横向滚动改为下拉换行布局，所有选项一目了然，点击"收起 ↑"恢复
- **订阅日程页**：tabBar"关于"改为"订阅日程"，提供三种订阅方式：
  - 微信提醒：在活动详情页点击"提醒我"
  - 日历订阅：分 iOS / Android 说明卡片，国产手机（华为/小米/OPPO）日历不支持订阅有专门提示
  - 网页版：复制 H5 链接在浏览器打开
- **活动提醒功能**：详情页新增"🔔 提醒我"按钮
  - 调用 `wx.requestSubscribeMessage` 请求微信订阅消息授权
  - 授权后本地记录，按钮变为"✅ 已设置提醒"
  - 支持取消提醒
  - 测试号降级处理（弹窗提示）
- **FAQ 常见问题**：订阅日程页底部新增常见问题解答

### 修复
- WXML 中 `wx:for` + `wx:if` 同元素冲突问题
- 条件 `bindtap` 不被支持的问题，改用 JS 内判断 disabled 状态
- WXML 不支持 `.substring()` 方法的问题，改为 JS 预处理数据

### 工程化
- 创建 GitHub 仓库：https://github.com/islon/goout-miniprogram
- 全部 26 个文件通过 GitHub API 推送至远程仓库

## v2.1 — 远程数据拉取（2026-07-11）

### 新增功能
- **运行时远程拉取数据**：app.js 启动时从远程获取最新 `exhibitions.json`
- **6 小时缓存**：本地缓存数据，6 小时内不重复请求，缓存过期后自动拉取
- **离线兜底**：网络不通时使用打包时内置的本地数据
- **加载动画**：数据加载中显示旋转 spinner + "正在加载活动数据..." 提示
- **下拉刷新**：首页下拉强制重新拉取远程数据

### 技术改动
- app.js 新增 `initData()` 方法，管理远程数据获取和缓存
- app.js 新增 `onReady()` 回调机制，页面等待数据准备完成后再渲染
- 首页和详情页改为从 `app.globalData.exhibitions` 获取数据

## v2.3 — jsDelivr CDN 加速（2026-07-11）

### 性能优化
- **数据源切换**：从 GitHub Pages 切换为 jsDelivr CDN
  - 原地址：`https://islon.github.io/goout/output/exhibitions.json`
  - 新地址：`https://cdn.jsdelivr.net/gh/islon/goout/output/exhibitions.json`
  - 国内访问速度大幅提升
- **时间戳绕缓存**：所有远程请求 URL 加 `?t=时间戳` 参数
  - 绕过 jsDelivr CDN 缓存
  - 确保用户点刷新一定能拿到 GitHub 最新数据
  - 解决"数据已更新但用户看不到"的问题

### 数据更新链路
```
开发者更新 GitHub 仓库的 exhibitions.json
        ↓
用户打开小程序（6小时内用本地缓存，超时自动拉取）
        ↓
用户点刷新按钮 → URL 加时间戳 → 绕过 CDN 缓存 → 直接拿 GitHub 最新数据
```

## v2.4 — 场馆指南与反馈入口（2026-07-14）

### 新增功能
- **场馆指南页**：tabBar 新增「场馆」入口，展示全部 395 个已收录场馆
  - 覆盖深圳、北京、上海、广州、杭州 5 个城市
  - 支持城市筛选、类型筛选、关键词搜索
  - 每个场馆展示名称、类型、地址、门票、亮点标签和近期活动数
  - 场馆卡片显示「暂无活动 · 场馆也值得逛」，满足「没有活动也想看看」的场景
- **场馆详情页**：从活动详情或场馆列表进入，展示场馆完整信息
  - 地址、交通、门票、场馆介绍、亮点标签
  - 支持复制官网链接和地址
  - 列出该场馆的近期活动，点击可跳转活动详情
- **活动详情页增加场馆介绍**：通过场馆名模糊匹配，自动展示场馆简介、地址、交通、亮点
  - 匹配不上时提供「查看场馆指南」和「反馈补充」入口
- **反馈入口**：新增「意见反馈」页
  - 微信官方 `open-type="feedback"` 提交给小程序管理员
  - 提供反馈分类（数据报错、功能建议、场馆补充、其他）
  - 支持复制整理好的反馈文本
  - 提供 GitHub Issues 链接复制，适合数据报错和追踪
- **订阅日程页**：增加场馆指南和反馈入口卡片，同步更新 5 城市活动与场馆数量

### 工程改动
- `app.js` 新增 `venue_info.json` 远程拉取 + 本地缓存 + 离线兜底
- 新增 `data/venues.js` 本地场馆兜底数据（395 条，约 216KB）
- `utils/helpers.js` 新增 `findVenue` 模糊匹配和 `buildVenueActivityCounts` 活动计数
- `data/filters.js` 增加杭州城市和相关区县
- 新增 `pages/venues`、`pages/venue`、`pages/feedback` 三个页面
- 生成「场馆」tab 图标 `assets/tab-venue.png` / `tab-venue-active.png`

### 修复与优化
- 数据源最终改用 GitHub raw 直连（`raw.githubusercontent.com`），避免 CDN 缓存滞后
- 远程数据同时拉取 `exhibitions.json` 和 `venue_info.json`，时间戳绕过缓存
