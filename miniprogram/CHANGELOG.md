## v1.0 — 初版发布（2026-07-11）

将 https://islon.github.io/goout/ H5 网页项目完整转换为微信小程序。

### 新增功能
- **活动列表页**：488 条活动数据，支持城市、时间、亲子、类型、区县、地点、费用共 7 个维度筛选
- **智能筛选**：无结果选项自动置灰，避免空筛选
- **搜索功能**：支持活动名称、地点关键词搜索
- **分页加载**：每页 20 条，支持上一页/下一页
- **活动详情页**：点击卡片跳转详情，展示完整活动信息和原始链接
- **关于页**：项目介绍、覆盖城市、日历订阅链接
- **日期标记**：今天/明天/即将开始的状态 Badge
- **亲子 & 费用标识**：亲子友好、免费/收费可视化标签

### 技术实现
- 数据内嵌为 JS 模块（488 条，约 320KB）
- 筛选逻辑完整复用原网页 JavaScript
- scroll-view 实现横向滚动筛选条
- 详情页通过 encodeURIComponent 传递活动 ID
- 沙色系（#D4A373）设计风格，与原网页保持一致

## v2.3 — jsDelivr CDN 加速（2026-07-11）

### 性能优化
- **数据源切换**：从 GitHub Pages 切换为 jsDelivr CDN
  - 原地址：`https://islon.github.io/goout/output/exhibitions.json`
  - 新地址：`https://cdn.jsdelivr.net/gh/islon/goout/output/exhibitions.json`
  - 国内访问速度大幅提升
- **时间戳绕缓存**：所有远程请求 URL 加 `?t=时间戳` 参数
  - 绕过 jsDelivr CDN 缓存
  - 确保用户点刷新一定能拿到 GitHub 最新数据
  - 解决"数据已更新但用户看不到"的问题

### 数据更新链路
```
开发者更新 GitHub 仓库的 exhibitions.json
        ↓
用户打开小程序（6小时内用本地缓存，超时自动拉取）
        ↓
用户点刷新按钮 → URL 加时间戳 → 绕过 CDN 缓存 → 直接拿 GitHub 最新数据
```

## v2.5 — 数据加载重构：三级分级 + 官方核实 + 订阅页精简（2026-07-15 ~ 07-16）

### 需求背景
- 首屏优先展示「当前最新活动」，历史活动最后加载，避免用户一进来看到老活动
- 线上没有人工核实环节，需改用「官方核实」替代，并在订阅页如实告知用户数据来源

### 新增 / 优化
- **三级分级加载**：Tier1 近期活动（每城进行中/即将开始前 80 条，首屏）→ Tier2 全量分城市文件（后台静默补全）→ Tier3 历史活动（最后拉）。缺失城市自动重试补齐，未全量则不标记"已完整"
- **官方核实替代人工核实**：`verify_activities.py` 每日带链接核实（200~399=已核实；404/410=存疑；超时=未核实但不删），详情页显示核实状态徽标；订阅页更新频率文案改为「每日自动抓取，来自官方和网络，请注意核实」
- **订阅页精简**：合并"覆盖城市"到城市日程列表（去除重复块）；删除常见问题 FAQ；反馈入口精简为「微信 feedback 按钮 + GitHub 卡片直链」
- **修复**：深圳长期显示老活动（Tier1 选取逻辑改为按进行中/即将开始）；北京/广州等 5 城永久卡 80 条（Tier2 失败静默丢弃→改为报告失败+自动补齐+partial 缓存）；武汉/西安个别城市卡 80（Tier3 误标 isPartial=false 污染缓存→改为按实际覆盖城市判定）

### 技术改动
- `app.js`：`fetchAllExhibitions` 返回 `{merged, failed}`；新增 `_fillMissingCities(failed, rounds)` 带退避重试；`isPartial`/`CACHE_PARTIAL_KEY` 残缺标记
- `app.js`：订阅页 `onDataUpdated` 改为每次 render 后重登记（一次性回调导致计数不刷新）

## v2.6 — 场馆指南全量加载（2026-07-16 ~ 07-18）

### 需求背景
- 场馆指南只显示 481 / 378 个（打包裁剪兜底集），真实场馆约 2964 个，需展示全量

### 新增 / 优化
- **分城市场馆并行加载**：`generate_city_venues.py` 把 `venue_info.json`（2964 条）拆分为 10 个 `venue_info_{城市}.json` 小文件，弱网更稳
- **单文件兜底**：分城市小文件整体失败（如 CDN 冷缓存）时，退一步拉全量 `venue_info.json` 一次补齐全量
- **数量校验兜底 + 启动保险**：拉到的场馆数 < 预期全量 90% 时强制单文件补全；启动 5 秒后若仍远少于全量再试一次，覆盖分城市请求慢/卡住的边界
- **不降级防护**：远程拉到的场馆数远少于当前（缓存）时不覆盖，避免全量被兜底集冲掉
- **场馆指南增强**：新增区县筛选 + 分类多行折叠展开（venues 页）

### 技术改动
- `app.js`：`fetchAllVenues` / `_fillMissingVenueCities` / `applyVenues(arr, isPartial)` / `VENUE_PARTIAL_KEY` / `fetchVenueFullFallback`
- CI：`generate_city_venues` 接入数据更新流水线

## v2.7 — 版本/时效展示 · 智能缓存 · 刷新动画（2026-07-18）

### 新增功能
- **关于页版本与时效展示**：底部显示小程序版本号 + 发布日期 + 「数据更新于 {时间}」，让用户了解数据实效性
- **云侧无更新不下载（智能缓存）**：新增 `output/data_meta.json`（数据内容哈希版本号）。小程序启动先拉这个小文件（约 200B）比对本地缓存版本：相同则复用本地缓存、不重复下载几 MB 分城市大文件（省流量、秒开）；不同才下载。手动刷新同理，云侧无新数据提示「已是最新」

- **首页自动刷新修复**：首页 `onDataUpdated` 改为每次渲染后重登记，后台分级加载/场馆补齐后列表自动刷新，不再灰屏需手动下拉

### 技术改动
- `app.js`：`_checkMeta()` 版本比对；`onLoadingChange` 全局加载状态订阅；`APP_VERSION` 升至 `2026.07.18.14`（清过期缓存）
- `scripts/generate_data_meta.py` 生成数据版本清单；CI 接入
- `pages/index/index.js` 订阅加载状态驱动图标旋转
